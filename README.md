# LockIn

**LockIn** is a JavaFX-based collaborative study platform with authentication, social posting, messaging, and voice communication features in one desktop app.

## Feature List

- Theme switching (Light/Dark)
- Password change
- Authentication (sign up, login, logout, token session)
- Posting
- Commenting
- Direct messaging
- Common chat
- Profile
- Image and file attachments
- Voice calling

## How to Run Server

The server entry point is `app.lockin.lockin.server.ServerManager`.

### Option 1: Run from IDE (recommended)

- Open the project in IntelliJ IDEA
- Run the `main` method in `ServerManager`
- The TCP server starts on port `5000` and UDP relay starts on port `5001`

### Option 2: Run from terminal

```bash
./gradlew classes
java -cp build/classes/java/main app.lockin.lockin.server.ServerManager
```

On Windows PowerShell:

```powershell
.\gradlew.bat classes
java -cp build\classes\java\main app.lockin.lockin.server.ServerManager
```

## Real IP vs Shared IP

### How to run server locally (same Wi-Fi)

- Start the server on the host machine
- Get host IP from `ipconfig` (IPv4 address)
- Use that IP in the client's `server.txt` (or default to localhost when running on same machine)

### Pinggy (make shared IP real-IP-like)

- Use Pinggy to expose your local server to outside networks
- Share the public host/port generated by Pinggy with clients
- Clients can connect by updating `server.txt`

### ZeroTier (simulate distant network as local)

- Join server and clients to the same ZeroTier network
- Use ZeroTier-assigned IP to connect, similar to LAN behavior
- Useful for testing cross-network usage with private addressing

## How to Run Client

The client entry point is `app.lockin.lockin.client.MyApplication`.

### Run from terminal

```bash
./gradlew run
```

On Windows PowerShell:

```powershell
.\gradlew.bat run
```

### How to change custom server address in `server.txt`

The client reads `server.txt` from your user folder at:

- Windows: `%USERPROFILE%\.lockin\server.txt`
- Linux/macOS: `~/.lockin/server.txt`

Use this format:

```text
<server_host>
<server_port>
```

Example:

```text
192.168.0.105
5000
```

If `server.txt` is missing or invalid, the client falls back to:

- Host: `localhost`
- Port: `5000`

## FAQ

### Why does client fail to connect?

- Check whether the server is running
- Verify IP/port in `server.txt`
- Ensure firewall allows TCP `5000` and UDP `5001`

### Do I need `server.txt` for local development?

No. If `server.txt` does not exist, client uses `localhost:5000`.
